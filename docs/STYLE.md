# Style and Verification Policy

This policy keeps report output readable in browsers and reliable for ingestion.

## Structure Rules

Use a consistent heading pattern in reports:

1. Model Releases
2. Revenue and Fiscal Signals
3. Infrastructure and Data Centers
4. Ecosystem and Policy

## Verification Rules

- Every non-trivial claim should include a source URL.
- Keep [UNVERIFIED] tags intact until corroboration is complete.
- Do not infer certainty from single-source social posts.

## Browser Readability Guidelines

- Keep sections short and scannable.
- Use plain markdown links instead of raw URLs when possible.
- Prefer bullet lists over dense prose for updates.
- Put highest-value updates first in each section.

## Ingestion Compatibility

- Keep heading names stable.
- Avoid changing file locations without updating docs links.
- Keep latest artifact path stable at [../reports/ai/latest.md](../reports/ai/latest.md).

## Editorial Checklist

- BLUF present near top
- Key updates grouped by section
- Sources present and visible
- [UNVERIFIED] tags preserved when applicable
